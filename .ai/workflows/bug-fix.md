# Bug Fix Workflow

> 버그 수정 시 따르는 표준 절차입니다.

## 절차

### 1. 이슈 파악

- 이슈 내용을 읽고 재현 조건을 확인한다
- 재현 가능하면 직접 재현하여 증상을 확인한다

### 2. Root Cause 분석

- 관련 코드를 읽고 원인을 파악한다
- 추측으로 수정하지 않는다 — 원인을 정확히 이해한 뒤 수정한다

### 3. 테스트 먼저 작성

- 버그를 재현하는 실패 테스트를 먼저 작성한다
- 이 테스트가 현재 코드에서 실패하는 것을 확인한다

### 4. 최소한의 수정

- 버그를 고치는 데 필요한 최소한의 변경만 한다
- 주변 코드를 리팩토링하거나 "개선"하지 않는다

### 5. 검증

- 작성한 테스트가 통과하는지 확인한다
- 기존 테스트가 깨지지 않았는지 전체 테스트를 실행한다

### 6. 커밋

- 커밋 메시지: `fix: <무엇을 고쳤는지 간결하게>`
- 관련 이슈가 있으면: `fix: <설명> (#이슈번호)`

## 예시

```bash
# 테스트 실행 후 실패 확인
npm test -- --grep "should handle empty input"

# 수정 후 테스트 통과 확인
npm test

# 커밋
git commit -m "fix: handle empty input in parseConfig (#42)"
```
