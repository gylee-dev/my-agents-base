# PR Review Workflow

> Pull Request 리뷰 시 따르는 표준 절차입니다.

## 리뷰 체크리스트

### 정확성

- [ ] 요구사항을 올바르게 구현했는가?
- [ ] 엣지 케이스를 처리했는가?
- [ ] 기존 기능이 깨지지 않았는가?

### 코드 품질

- [ ] AGENTS.md의 코드 스타일을 따르는가?
- [ ] 불필요한 복잡성이 없는가?
- [ ] 중복 코드가 없는가?
- [ ] 함수/변수 이름이 명확한가?

### 테스트

- [ ] 적절한 테스트가 포함되어 있는가?
- [ ] 테스트가 의미 있는 동작을 검증하는가?
- [ ] 모든 테스트가 통과하는가?

### 보안

- [ ] 하드코딩된 비밀 값이 없는가?
- [ ] 사용자 입력을 적절히 검증하는가?
- [ ] SQL injection, XSS 등 취약점이 없는가?

### Git

- [ ] 커밋 메시지가 Conventional Commits를 따르는가?
- [ ] PR이 하나의 관심사만 다루는가?

## 리뷰 코멘트 작성 가이드

- 문제가 있으면 **왜** 문제인지 설명하고 대안을 제시한다
- 사소한 스타일 지적보다 로직과 설계에 집중한다
- 좋은 코드에는 칭찬 코멘트를 남긴다
